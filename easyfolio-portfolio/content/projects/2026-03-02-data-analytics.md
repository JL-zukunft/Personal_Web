## 为什么做？
![[2026-05-01-peronal-aihtml-20260526--1.png]]
说白了，我就是觉得现有的 Markdown 预览太死板了，全是整齐划一的方块字，一点“呼吸感”都没有。我想让每一个加粗的句子、每一行列表都能像精美的档案卡片一样跳出来，让阅读项目案例变成一种在档案室翻阅秘密卷宗的体验。

**我其实特别反感那种正式的项目报告腔，总觉得要把折腾的过程写得有血有肉才对。所以这不仅仅是为了测试样式，更是为了验证一种“叙事美学”在网页上能不能立得住。**

我觉得一个好的作品集，不应该只是展示结果，更应该通过视觉节奏，让别人看到我当时纠结和兴奋的瞬间。私心喜欢那种带有工业档案感的质感，所以这次测试也是为了给自己的 vibe coding 找个完美的载体。

## 满足了什么？

这个测试文档补齐了从“原始纯文本”到“工业感 UI”之间的那个断层。它满足了我对页面“秩序感”的执着，让普通的 Markdown 语法通过 CSS 的魔改，自动生成带有左侧竖线、独立卡片和步骤条的视觉结构，不需要我再去手写复杂的 HTML。

## 如何达成？

我的目标是让用户（也就是我自己）即使只写简单的 MD，也能得到设计师级别排版的效果。

设计上我参考了一些优秀的工业风模板，特别是《明日方舟》那种克制且冷峻的质感，颜色选用了我最爱的 Sage（鼠尾草绿）配合 Cream（奶油色），既专业又不会太刺眼。

技术栈如下表：

| 模块 | 技术 / 工具 | 作用 |
| :--- | :--- | :--- |
| 渲染底层 | JavaScript Regex | 核心解析逻辑，负责把 MD 标签转为 UI 块 |
| 视觉风格 | CSS Variables | 统一定义 Sage & Cream 工业色谱 |
| 字体方案 | Google Fonts | 引入 Syne 与 JetBrains Mono 增加专业感 |
| 协作工具 | Trae / Claude | 辅助定位正则冲突，解释复杂的样式覆盖问题 |

AI 工具链在这次“样式实验室”里帮了大忙。我其实在写复杂的正则表达式时挺小白的，经常把章节标识给切断了，多亏了 AI 帮我“标注、解释、定位”，我才搞明白为什么内容会突然消失，然后针对性地修补了换行逻辑。

## 克服的困难与挑战？

说实话，刚开始搞这个解析器时我真的挺挫败的。最让我头大的不是美化，而是“间距”——MD 自动生成的换行符和我的卡片外边距叠加在一起，让页面看起来空得能跑马。

- **信心不足**：我连 `transform` 和 `translate` 有时候都分不清，更别说处理样式层级的覆盖了。
- **定位 Bug**：有时候内容突然识别失败，我一度以为是 JS 坏了，后来让 AI 帮我分析，才发现是因为 HTML 标签被正则表达式暴力截断了。
- **视觉微调**：为了让 Callout 块看起来像一叠有序的卡片，我微调了不下 20 次 `margin` 参数，直到它们挨得刚刚好。

1. 学习让 AI 帮我逐行解释代码逻辑，定位样式覆盖问题。
2. 重新重构了 `formatText` 内部的正则顺序，确保块级元素优先识别。
3. 耗时最长的就是这个节点地图的点击跳转，要确保它能精准对应到生成的 `chapter-id`。
